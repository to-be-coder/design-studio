# Studio house language

**The idea.** A studio-default `DESIGN.md` — the studio's own type, spacing, and color sensibility
plus its accumulated Do's and Don'ts — used as the **fork base for greenfield and unbranded
projects**. Client-brand work ignores it; everything else stops defaulting to generic-AI
aesthetics and starts defaulting to the studio's taste.

**Why it was cut.** Deserves its own deliberate session rather than being a side effect of
pipeline work. V1 shipped only the hook: the `design-system` skill offers a studio-default
DESIGN.md as fork base when one exists.

**What it needs to live**
- A quiet week, and the pleasing move of running it through the pipeline itself as a Lite project:
  "design the studio's visual language."
- A few harvested `applies: taste` pages to draw from — the house language should be *distilled
  from real project taste calls*, not invented in a vacuum.
- A decision on where it lives (likely beside the starter wiki, versioned with the studio's own
  vault).

*Note for shipped copies: this spark is intentionally a taste-class example — it shows the format,
but the house language it describes must be **yours**. Delete this card when your own version
exists.*
