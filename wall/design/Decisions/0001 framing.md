# 0001 — The wall is a proof artifact first, a tool second

**Decision.** Frame Studio Wall as the visible face and control surface of the design-studio
OS, whose primary job is to demonstrate the product's thesis (visual contract ⇒ designed output)
while being a genuinely useful ambient display and skill runner. Product feature from day one.

**Why.** The dashboard genre is saturated with neon demo-ware; the differentiating asset the
studio owns is the pipeline itself. A dashboard built *through* the pipeline, with its design
record public, converts the product's method into its marketing. Laddered to the guiding
principle: the artifact must prove the claim it makes.

**Rejected alternatives.**
- Personal-first, productize later — rejected by the user in debrief Q&A (product feature now).
- Read-only wall — superseded by [[0002 control-surface]].
- Neon/multi-accent aesthetic — out of scope by brief ("one color").

**True cost.** build: one prototype (server + static UI + DESIGN.md) / support: a real product
surface users will file issues against — accepted knowingly.
